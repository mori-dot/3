# 亚马逊跨境电商运营个人官网

这是一个为新手小白设计的亚马逊跨境电商运营个人官网，使用纯HTML+CSS+简单JS开发，无需任何框架，易于上手和部署。

## 网站结构

网站包含以下5个页面：

1. **首页**：顶部导航 + 姓名-职业+核心一句话+联系按钮 + 极简底部
2. **关于我**：页面标题+个人简介+核心优势+ 极简底部
3. **核心服务**：页面标题+服务模块+ 极简底部
4. **案例展示**：页面标题+Listing截图展示+ 隐私备注+ 极简底部
5. **联系方式**：页面标题+微信/电话/邮箱+ 极简底部

## 如何使用

### 1. 填充内容

打开以下文件，将标记为【】的内容替换为你自己的信息：

- `index.html`：首页的姓名、核心一句话
- `about.html`：个人简介、核心优势
- `services.html`：各服务模块的内容
- `cases.html`：隐私备注
- `contact.html`：微信、电话、邮箱

### 2. 上传图片

1. **个人职业照**：将你的职业照命名为 `profile.jpg`，放入 `images` 文件夹
2. **案例图片**：将你的亚马逊产品Listing截图命名为 `case1.jpg`、`case2.jpg`、`case3.jpg`、`case4.jpg`，放入 `images` 文件夹

### 3. 图片要求

- **格式**：推荐使用 JPG 或 PNG 格式
- **大小**：职业照建议200x200像素，案例图片建议800x600像素左右
- **文件大小**：单张图片建议不超过2MB，确保页面加载速度

## GitHub部署教程（零基础小白版）

### 步骤1：注册GitHub账号

1. 打开 [GitHub官网](https://github.com)
2. 点击右上角的 "Sign up"
3. 填写用户名、邮箱和密码
4. 按照提示完成注册流程

### 步骤2：创建仓库

1. 登录GitHub后，点击右上角的 "+" 号，选择 "New repository"
2. 仓库名称填写：`你的用户名.github.io`（例如：`zhangsan.github.io`）
3. 仓库描述：可以填写 "亚马逊跨境电商运营个人官网"
4. 选择 "Public"（公开）
5. 勾选 "Add a README file"（可选）
6. 点击 "Create repository"

### 步骤3：上传文件

1. 进入刚创建的仓库页面
2. 点击 "Add file" -> "Upload files"
3. 点击 "choose your files" 或直接拖拽文件到页面
4. 选择本地 `个人官网2` 文件夹中的所有文件和文件夹（包括：index.html、about.html、services.html、cases.html、contact.html、style.css、script.js、images文件夹）
5. 等待文件上传完成
6. 在页面底部 "Commit changes" 处填写提交信息，例如："Initial commit"
7. 点击 "Commit changes"

### 步骤4：开启GitHub Pages

1. 在仓库页面，点击 "Settings"
2. 在左侧菜单中点击 "Pages"
3. 在 "Source" 选项中，选择 "main" 分支（或 "master" 分支，取决于你的默认分支名称）
4. 点击 "Save"
5. 等待几分钟，页面会显示你的网站访问链接，例如：`https://你的用户名.github.io`

### 步骤5：访问你的网站

打开浏览器，输入你的GitHub Pages链接，例如：`https://zhangsan.github.io`，即可访问你的个人官网。

## 常见问题及解决方法

### 1. 图片不显示

- **原因**：图片路径错误或图片未上传
- **解决方法**：
  - 确保图片已正确放入 `images` 文件夹
  - 确保图片文件名与代码中的引用一致（例如：`profile.jpg`、`case1.jpg` 等）
  - 检查GitHub仓库中是否成功上传了 `images` 文件夹

### 2. 网站样式错乱

- **原因**：CSS文件未正确上传或路径错误
- **解决方法**：
  - 确保 `style.css` 文件已成功上传到GitHub仓库
  - 检查HTML文件中是否正确引用了CSS文件：`<link rel="stylesheet" href="style.css">`

### 3. 网站无法访问

- **原因**：GitHub Pages未完全部署或仓库名称错误
- **解决方法**：
  - 确保仓库名称格式正确：`你的用户名.github.io`
  - 等待几分钟，GitHub Pages需要时间部署
  - 检查仓库设置中的Pages配置是否正确

### 4. 页面响应式问题

- **原因**：浏览器缓存或设备兼容性
- **解决方法**：
  - 清除浏览器缓存后刷新页面
  - 尝试在不同设备上测试网站
  - 确保图片尺寸合适，避免过大影响加载速度
